Multi-Map Master Reference

Welcome to the definitive documentation for Multi-Map v14.11. This wiki serves both daily users navigating the spatial interface and developers building federated companion applications.

Part I

User Guide

1. Getting Started

Multi-Map is a spatial mapping ecosystem. Instead of folders and files, you organize information into "Constellations" of interconnected nodes. Every map begins with a Universe Root. From there, you can spawn Hubs, Notes, Portals, and structured schema elements depending on the Map's designated Type.

2. Navigation & Controls

Mouse & Touch Controls

  • Pan: Click and drag the background.
  • Zoom: Scroll wheel or Pinch-to-zoom.
  • Select Node: Click a node to open its Radial Menu.
  • Move Node: Click and drag a node to pin it manually.

Global Search: Use the navigation search bar to fuzzy-search across node titles, types, and content. Selecting a search result will automatically uncollapse its parent branch and glide the camera to the node.

Keyboard Power-User Shortcuts

Multi-Map natively supports standard mind-map keyboard editing for rapid, mouse-free node creation. When a node is selected:

Shortcut Action
Tab Creates a new Child node and focuses the Inspector text entry.
Enter Creates a new Sibling node (same type) and focuses text entry.
Space Triggers the node's primary Smart Action (e.g., Run Chain, Launch Portal).
Arrows (←↑↓→) Semantic Traversal: intelligently navigates up to parents, down to closest children, or across siblings based on layout geometry.
Ctrl/Cmd + C / V Copy and paste entire node branches (even across maps). You can also paste formatted indented text to auto-generate node trees!

3. Node Operations & Reparenting

Targeted Linking

To connect two existing nodes without making them parent/child, click the Link (🔗) button on the Radial Menu. The system enters Linking Mode (pulsing blue UI). Navigate to your target node and click the green Confirm Link button to establish a dashed association line.

Changing Parents (Reparenting)

To move a node to a different branch, you can either:

  • Click the Select Parent (👆) button in the radial menu.
  • Shortcut: Alt + Click the node you want to move.

The screen will show a pulsing overlay. Next, simply click the new parent node and select the green Confirm button. Schema rules are strictly enforced (e.g., a node cannot be its own child, nor can a portal have children).

Clipping Branches

If a map becomes too cluttered, select the Clip (✂️) action on a node. This deletes the node and its downstream children from the current map, pastes them into an entirely new Map file, and leaves behind a Portal node in its exact place pointing to the new file.

4. Phase Engines (Views)

Multi-Map separates your raw data from how it is displayed. "Phases" are different iframe-based lenses through which you can view the exact same Mapstate JSON.

  • 🗺️
    Celestial Map

    The default spatial graph. Free-form dragging, force-directed physics engine enabled.

  • 🌐
    Web Architect

    Dynamically compiles `web-*` nodes into live HTML/Tailwind inside an iframe. Capable of mounting live URLs via the web-root node.

  • 📜
    Prompt Engine

    Compiles downstream `prompt-*` nodes (role, context, goal, instructions) into a linear, formatted Markdown document for LLM consumption.

  • 🤖
    Agent Config

    Dedicated sandbox viewport providing a dynamic overview of the agent architecture (skills, tools, memory).

  • 👤
    Person / Identity

    Translates `person-root` node data into a professional profile or CV format.

5. The Project Directory & Folders

Multi-Map organizes files hierarchically into Projects, Folders, and Pages via the Data Manager sidebar.

📁 The Master Map

Every project automatically generates a special Project Directory page. This acts as the "Master Map" for the project.

  • It visually represents your project's architecture using a `file-root` structure.
  • Bi-directional Sync: Moving a map portal into a folder node on the Canvas automatically updates the folder hierarchy in the left-hand Data Manager UI, and vice-versa.
  • One per Project: A project can strictly only have one master directory.
  • Quick Access: From any map, selecting the "Project Directory (🏠)" button on a root node instantly returns you to the Master Map.

6. Smart Roots & Portals

To link maps together, create a Portal node. Portals act as wormholes to other pages in your library.

Dynamic Portal System

In previous versions, a specialized `smart-portal` node type existed. This has been deprecated. Now, all Portals are dynamic. The system utilizes Smart Roots to determine behavior.

  1. Every Root node contains standardized root_metadata.
  2. When you interact with a Portal, it peeks at the target map's root metadata.
  3. openPortal(): If configured, interacting with the portal automatically spawns child data natively within the current map based on the target map's payload (e.g., executing a prompt chain and returning the result locally).
  4. enterPortal(): The standard action replaces your current workspace with the destination map. The system maintains a "Portal History" stack allowing you to safely "Exit Portal" back up the chain.

7. MM-AI Assistant

Multi-Map features a native LLM integration (Gemini) capable of converting natural language into spatial MapState JSON, modifying existing maps, and providing structural tutoring.

✨ Autocomplete Commands

  • /generate - Create a completely new map or branch.
  • /edit - Modify the current graph structurally.
  • /project - Generate a multi-page project package.
  • /explain - Narrate the current map context.

🗣️ Local Sandbox Commands

The AI can interpret commands locally without hitting the cloud API. Try commands like:
"Select node Alpha", "Rename node X to Y", "Delete node Z", or "Link Alpha to Beta".

Part II

Developer Documentation

8. Core Architecture

The platform relies on a Decoupled State Machine pattern.

The Kernel (multi-map-core.js)

Holds the MapState object. Performs array mutations, physics loops, handles syncing logic with Firestore, and enforces the "Project Directory" bidirectional folder-to-node synchronization via syncProjectMasterMap().

The Sandbox Controller (multi-map-sandbox.js)

Handles DOM events, viewport transforms, UI overlay logic, Radial Menu generation, and keyboard navigation routing.

10. Force-Directed Physics

The layout is driven by a custom iterative algorithm found in Kernel.resolveOverlaps().

1. Universal Repulsion

An O(N²) calculation where every node pushes away if distance < 240px. (kRepel = 0.08).

2. Hooke's Law Springs

Only applies to structural connections. Pulls connected nodes toward an ideal resting distance of 160px. (kSpring = 0.05).

11. Federation & API Roadmap

The platform is actively transitioning toward a federated, decentralized storage model.

Bring Your Own Storage (BYOS)

Currently, maps are saved to localStorage or a centralized Firebase Cloud Vault. The map schema includes a storage_target property preparing for Google Drive and Local OS integration via the HTML5 File System API.

True Federation (WebRTC + CRDTs)

Upcoming releases will transition live collaborative edits from Firestore onSnapshot listeners to direct P2P WebRTC data channels, utilizing CRDT frameworks (like Yjs or Automerge) to handle conflict resolution across community `colab-root` spaces.
